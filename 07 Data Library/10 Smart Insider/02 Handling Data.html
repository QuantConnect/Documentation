There are over 40 fields available for each Smart Insider data type for use in your algorithms. Data is published in UTC and is updated daily, so be sure to set your resolution to Resolution.Daily.

You can view the fields available for use in your algorithm by clicking on the type names below:

  • $[SmartInsiderEvent,T:QuantConnect.Data.Custom.SmartInsider] - These fields are available in both intentions and transactions types
  • $[SmartInsiderIntention,T:QuantConnect.Data.Custom.SmartInsider]
  • $[SmartInsiderTransaction,T:QuantConnect.Data.Custom.SmartInsider]

To import the data types to your algorithm, you can do the following: 

using QuantConnect.Data.Custom.SmartInsider;
        
from QuantConnect.Data.Custom.SmartInsider import *

AddData accepts an equity ticker value for Smart Insider data. For the data to be added properly, ensure that the ticker being added is the most up to date ticker as of today.

Note: Smart Insider data can undergo rename and delisting events because of its relationship to equities.

Note: Ensure you set the data timezone to UTC in AddData

If a TransactionID is duplicated in either SmartInsiderIntention or SmartInsiderTransaction, that means the record was amended and the old record should be considered outdated.

Smart Insider data is provided in an event-sourced/delta format. Transaction and intention events are transmitted as they were at a specific point in time. Transactions can have corrections issued to them at a future point in time after the original data has been reported. An example of this would be receiving a piece of data on 2019-01-01 with the transaction ID TX-1234, and then receiving another piece of data on 2019-02-05 with the same transaction ID TX-1234. The event reported on 2019-02-05 should be considered as the most current data, and the old event from 2019-01-01 should be discarded. An example on how to detect these events is provided below: 

public class SmartInsiderAlgorithm : QCAlgorithm 
{
    private HashSet<string> _transactionIDs = new HashSet<string>();
    private Symbol _symbol;

    public override void Initialize()
    {
        // ...
        _symbol = AddData<SmartInsiderTransaction>("AAPL", Resolution.Daily, TimeZones.Utc).Symbol;
        // ...
    }

    public override void OnData(Slice data)
    {
        if (!data.ContainsKey(_symbol)) 
        {
            return;
        }

        var transactionID = data[_symbol].TransactionID;
        var isAmendment = _transactionIDs.Contains(transactionID);

        if (isAmendment)
        {
            Debug($"{Time} - Encountered duplicate event for ticker {_symbol.Value}");
        }
        
        // Add the transaction ID to the transactions ID store
        _transactionIDs.Add(transactionID);

        // ...
    }
}
        
class SmartInsiderAlgorithm(QCAlgorithm):
    def Initialize(self):
        # ...
        self.transactionIDs = []
        self.symbol = self.AddData(SmartInsiderTransaction, "AAPL", Resolution.Daily, TimeZones.Utc).Symbol
        # ...

    def OnData(self, data):
        if not data.ContainsKey(self.symbol):
            return
        
        transactionID = data[self.symbol].TransactionID
        addToTransactionsDatabase = True

        isAmendment = transactionID in self.transactionIDs

        if isAmendment:
            self.Debug(f"{str(self.Time)} - Encountered duplicate event for ticker {self.symbol.Value}")
            addToTransactionsDatabase = False
        
        # Add the transaction ID to the transaction ID store 
        if addToTransactionsDatabase:
            self.transactionIDs.append(transactionID)
        
        # ...

Reasons why transactions might be amended are:
Reason Description
Reporting errors by the issuer The issuer of the report might have corrections
Reporting errors by Smart Insider Rare, but can occur
New information If additional information is received in the future regarding a transaction
Company change ISIN or local market ID changes

Transactions might be reissued with PriceAdjustedFactor and AmountAdjustedFactorfields in the case of:

  • Splits
  • Reverse splits
  • Special dividends (amount factor will be 1)
  • Spin-offs (amount factor will be 1)

Sometimes, in rare cases, Smart Insider might delete a transaction file and report it again, sometimes due to a duplicate data error, or misc. error. When a deletion occurs, the transaction is re-sent with the TransactionID, but all other fields null. The NoteText field will contain the word "DELETED" and may also contain an explanation of why the data was deleted.

Please note that with the exception of the following fields, all fields have the potential to be null in both $[SmartInsiderIntention,T:QuantConnect.Data.Custom.SmartInsider] and $[SmartInsiderTransaction,T:QuantConnect.Data.Custom.SmartInsider]

  • TransactionID
  • LastUpdate
