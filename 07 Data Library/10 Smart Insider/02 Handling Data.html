There are over 40 fields available for each Smart Insider data type for use in your algorithms. Data is published in UTC and is updated daily, so be sure to set your resolution to Resolution.Daily.

For Smart Insider, AddData accepts a ticker for an equity for the ticker field.

Note: Smart Insider data can undergo rename and delisting events because of its relationship to equities. To learn more about handling delisting/rename events, visit Handling Data - SymbolChangedEvent

Note: Ensure you set the data timezone to UTC in AddData

Smart Insider data is provided in an event-sourced/delta format. What this means is that you can receive events as encountered at a specific point in time, but have a correction be issued after the fact invalidating that record, meaning you would have to replace the old data with the new. For more information about event sourcing/delta file formatting, visit https://docs.microsoft.com/en-us/azure/architecture/patterns/event-sourcing

If a TransactionID is duplicated, that means the record was amended and the old transaction should be considered outdated.

Reasons why transactions might be amended are:
Reason Description
Reporting errors by the issuer The issuer of the report might have corrections
Reporting errors by Smart Insider Rare, but can occur
New information If additional information is received in the future regarding a transaction
Company change ISIN or local market ID changes

Transactions might be reissued with PriceAdjustedFactor and AmountAdjustedFactorfields in the case of:

  • Splits
  • Reverse splits
  • Special dividends (amount factor will be 1)
  • Spin-offs (amount factor will be 1)

Sometimes, in rare cases, Smart Insider might delete a transaction file and report it again, sometimes due to a duplicate data error, or misc. error.

When a deletion occurs, the transaction is resent with the TransactionID, but all other fields null. The NoteText field will contain the word "DELETED" and may also contain an explanation of why the data was deleted.

Please note that with the exception of the following fields, all fields have the potential to be null in both $[SmartInsiderIntention,T:QuantConnect.Data.Custom.SmartInsider] and $[SmartInsiderTransaction,T:QuantConnect.Data.Custom.SmartInsider].

  • TransactionID
  • LastUpdate
