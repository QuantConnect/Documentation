The data is stored as a flat JSON file on disk. In the processing stage, we chose to preserve as many fields as possible without modifying the order in which data is represented. This way, we can preserve the structure and essence of the raw data in a structured format.

To access SEC data, you'll need to use one or more of the following data types:
Form Type Data Type
8-K $[SECReport8K,T:QuantConnect.Common.Data.Custom.SEC.SECReport8K]
10-K $[SECReport10K,T:QuantConnect.Common.Data.Custom.SEC.SECReport10K]
10-Q $[SECReport10Q,T:QuantConnect.Common.Data.Custom.SEC.SECReport10Q]

The data is accessible from the derived BaseData instances (SECReport8K, SECReport10K, etc.) under the class field "Report". The "Report" field is of type $[SECReport,T:QuantConnect.Common.Data.Custom.SEC.SECReport] and contains all of the report metadata in a structured format. The report documents are left in their original order and are unprocessed.

Note: Since we use the same interface in our BaseData derived classes (ISECReport), we'll use SECReport8K as an example to cover for both SECReport10K and SECReport10Q.

Taking a look at $[SECReport8K,T:QuantConnect.Common.Data.Custom.SEC.SECReport8K], we can see that there is a field that contains a type called $[SECReportSubmission,T:QuantConnect.Common.Data.Custom.SEC.SECReportSubmission]. It is the class containing all SEC metadata and document contents. To begin accessing SEC data, you'll need to retrieve your SECReport8K, SECReport10K, SECReport10Q, etc. from the Slice object, and then access the "Report" field: 

public void OnData(Slice slice)
{
    // ...
    // Save in two different variables in the case that we need to access the Symbol object
    var reportData = slice[_symbol];
    var report = reportData.Report;
}
        
def OnData(slice):
    # ...
    # Save in two different variables in the case that we need to access the Symbol object
    reportData = slice[self.symbol]
    report = reportData.Report

For $[SECReportSubmission,T:QuantConnect.Common.Data.Custom.SEC.SECReportSubmission], we describe subtle behaviors present for each field:
Field Remarks
AccessionNumber Rarely used, mostly as a way to reference the specific original filing on EDGAR
FormType Value is hyphenated
PublicDocumentCount Chosen as string because of potential non-number values
Period
Items Items are a good way to identify what kind of contents an 8-K report contains
FilingDate This is sometimes before the date that the data was released on
FilingDateChange Optional field. Does not always appear in reports. Check for 0001-01-01 to filter
MadeAvailableAt This is a field added by QuantConnect to identify when the data first became available for reality modeling purposes
Filers First filer is usually the company the report is for. Will always contain at least a single value
Documents Will always contain at least a single value

Focusing on the SEC report contents, we can access the document(s) by iterating on the Documents field. Inside the $[SECReportDocument,T:QuantConnect.Common.Data.Custom.SEC.SECReportDocument] object, the Text field contains all of the raw report contents.

Note: If the first line starts with "<XML>" (without double quotes), the contents will be in XML (except the first line), and can potentially be parsed with an XBRL parser
