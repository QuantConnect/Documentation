To connect the Cascade agent in the Windsurf Editor to the QC MCP Server, follow these steps:

  1. Install and open the Windsurf Editor.
  2. In the top navigation bar of Windsurf, click View > Extensions.
  3. In the Search Exensions in Marketplace field, enter QuantConnect.
  4. Click Install.
  5. In the left navigation bar, click the QuantConnect icon.
  6. In the Initialization Checklist panel, click Login to QuantConnect.
  7. In the Windsurf window, click Open.
  8. On the Code Extension Login page, click Grant Access.
  9. In the Select Workspace panel, click Pull Organization Workspace.
  10. In the Pull QuantConnect Organization Workspace window, click the cloud workspace (organization) that you want to pull.
  11. In the Pull QuantConnect Organization Workspace window, create a directory to serve as the organization workspace and then click Select.

    12. It takes a few minutes to create a new organization workspace directory and populate it with the the initial file structure. After the organization workspace is populated with the initial file structure, it pulls your cloud project files.

  12. In your ~ / .codeium / windsurf / mcp_config.json file, add the following configuration:
    13. {
  "mcpServers": {
    "qc-mcp": {
      "disabled": false,
      "url": "http://localhost:3001/"
    }
  }
}
  13. In Windsurf, create a new project or open an existing one.
  14. In the top navigation menu, click File > Preferences > Windsurf Settings.
  15. On the Windsurf Settings page, in the Cascade section, click Open MCP Marketplace.
  16. If the QC MCP Server has an error, click the refresh icon to restart it.
  17. Press Ctrl+L to open Cascade and begin interacting with the agent.

To backtest and live trade with the local Deployment Target, install Docker Desktop. The installation process depends on if your machine uses Windows, macOS, or Linux.
