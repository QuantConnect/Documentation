Memory lets an assistant remember facts about you and your organization across conversations and projects, so you don't restate the same context every time you start a task. You might record your default data resolution, the benchmark you compare against, or the asset classes your team trades.

Custom memory is a single memory.md file inside the reserved .assistant folder of your organization's Object Store (.assistant / memories / memory.md). It is plain Markdown, so you can write the facts as a simple list or as short notes. For example: 

- Default to hourly resolution for equity strategies unless asked otherwise.
- Benchmark all backtests against SPY.
- The team trades US Equities and Equity Options only.

Assistants that use memory load this file automatically at the start of each run, so your facts are always in context without any action from you. When you edit memory.md by hand, the change takes effect the next time an assistant runs. Assistants also keep the file current on their own, recording new facts they learn as they work, so their memory builds up over time.
