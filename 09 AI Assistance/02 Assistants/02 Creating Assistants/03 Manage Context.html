Context is everything the model sees when it runs: the system prompt, the task prompt, results from tool calls, and any data passed in. Every word counts against the model's context budget. Long contexts cost more, slow the model down, and crowd out the actual reasoning it needs to do. The goal is to give the assistant the minimum it needs to do the work well.

Keep the system prompt tight. Focus on the parts that change outcomes: the specific steps, the named tools, the hard constraints, and the format of the output. Avoid motivational language, repeated reminders, and paragraphs of background the assistant does not need. If you find yourself listing edge cases the assistant rarely sees, move that logic into the tool that handles the case instead of bloating the prompt.

Keep task prompts focused on the specific work for this run. Include the inputs the assistant needs and the constraints unique to this deployment, and don't repeat what is already in the system prompt.
