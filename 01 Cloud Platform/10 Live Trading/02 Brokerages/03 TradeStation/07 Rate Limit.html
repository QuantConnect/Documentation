TradeStation enforces rate limits on its REST API. The following table shows the request quota for each resource group over a rolling time window:

Resource Quota Window
Accounts, order details, balances, and positions 320 5 minutes
Quote snapshots 500 5 minutes
Each Option endpoint 90 1 minute

Streaming endpoints have concurrent-connection limits but don't consume the request quota. For more information about the rate limits, see Rate Limiting in the TradeStation documentation.

To respect these limits, our integration manages the request rate for you in two ways:

  • Proactive throttling: A per-resource-group token bucket paces requests to stay under the quota.
  • Reactive back-off: If a request receives an HTTP 429 (Too Many Requests) response, the integration waits for the period the X-RateLimit-Reset (or Retry-After) header specifies and then retries the request instead of failing the operation.

The first time the integration throttles a resource group or receives a 429 response, it sends your algorithm a one-time warning so you know throttling may reduce your throughput.
